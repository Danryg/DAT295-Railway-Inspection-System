# DAT295-Railway-Inspection-System

## Workflow

### Branches

Create a new branch for each new feature/fix you develop, and name it `feat/new-feature`/ `bug/bug-fix` etc. Once your feature is implemented, create a new merge request and assign atleast 1 reviewers.

#### Protection rules

The default(main) branch is protected, meaning that you can't push directly to it. You have to create a new
branch and create a merge request to the main branch. The merge request has to be approved by atleast 1 reviewers
before it can be merged.
